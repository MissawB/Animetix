# 🕸️ Design Spec: Animetix Localized Graph Visualizer (Phase 1)

**Date:** 2026-05-26  
**Status:** Approved  
**Topic:** Interactive Knowledge Graph exploration for Premium users.  
**Roadmap Context:** Phase 1 of the "Expansion of Animetix" (Monetization Extension).

---

## 1. Executive Summary
The **Localized Graph Visualizer** is a Premium feature that allows users to explore the hidden connections between media items (Anime, Manga, Games) and their thematic tropes using the existing Neo4j Knowledge Graph. It provides a radial, interactive "Mind Map" view centered on a specific work, with dynamic depth exploration.

---

## 2. User Experience (UX)

### 2.1 Entry Point
- A prominent **"Explore Connections"** (Explorer les connexions) button on every Media Detail page (frontend).
- Only visible/active for **Premium** tier users (verified via `request.user_tier`).

### 2.2 Core Interaction
- **Radial Layout:** The target media item is at the center. Satellite nodes represent related entities (Themes, Authors, Studios, other Media).
- **Dynamic Depth:** A slider allows users to expand the exploration range:
    - **Depth 1:** Direct connections only.
    - **Depth 2:** Neighbors of neighbors (The "Ecosystem").
    - **Depth 3:** Extended galaxy of connections.
- **Node Navigation:** Clicking any satellite node re-centers the entire graph on that node and fetches its neighborhood.
- **AI Interpretations:** Hovering over an edge (link) displays a tooltip generated by the AI Oracle explaining *why* these two items are connected.

---

## 3. Technical Architecture

### 3.1 Backend (Django & Neo4j)
- **New Port/Adapter Method:** Extend `Neo4jGraphAdapter` with `get_neighborhood(item_id, type, depth)`.
- **API Endpoint:** `GET /api/v1/graph/neighbors/`
    - Logic: Validate Premium status -> Call Neo4j using optimized Cypher (APOC) -> Return JSON `{nodes, links}`.
- **Cypher Strategy:**
    ```cypher
    MATCH (start {external_id: $id, type: $type})
    CALL apoc.path.subgraphAll(start, {maxLevel: $depth})
    YIELD nodes, relationships
    RETURN nodes, relationships
    ```

### 3.2 Frontend (React)
- **Library:** `react-force-graph-2d` for high-performance physics-based rendering.
- **State Management:** A custom hook `useGraphExplorer` to manage node/link state, deduplication, and lazy loading.
- **UI Components:**
    - `GraphModal`: Fullscreen container.
    - `GraphCanvas`: The interactive visualization.
    - `DepthSlider`: Range input for 1-3 sauts.

### 3.3 AI Integration
- **Oracle Link Explainer:** A dedicated prompt in `PromptManager` for explaining graph relationships.
- **Caching:** AI-generated explanations are stored in `SemanticCache` to ensure near-instant hover feedback for common links.

---

## 4. Performance & Scalability
- **Lazy Loading:** Increasing depth only fetches the *new* nodes/links from the backend.
- **Caching:** Subgraph results are cached in Redis for 24h.
- **Safety Limits:** Hard limit of 200 nodes per view to prevent browser performance degradation.

---

## 5. Success Criteria
- **Engagement:** Premium users spend at least 15% of their session time in the graph explorer.
- **Discovery:** Users discover at least 2 new titles per graph session via connections.
- **Backend Latency:** Neo4j neighborhood retrieval < 300ms for depth 2.
