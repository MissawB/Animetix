# Design Spec: Gold Set Architectural Improvement

**Date:** 2026-05-16
**Status:** Draft
**Topic:** Improving the MLOps Gold Set with Architecture-Specific Queries (Graph & Visual)

## 1. Problem Statement
The current `gold_dataset.json` is primarily focused on simple semantic retrieval for popular anime. It does not adequately benchmark the advanced capabilities of the Animetix architecture:
- **Neo4j GraphRAG**: Multi-hop relations, cross-media links, and metadata-rich contexts.
- **SigLIP Multimodal**: Visual attribute search and vibe-based alignment.
- **Agentic Logic**: Handling complex reasoning, ambiguity, and negative cases.

## 2. Proposed Solution
We will manually add a set of "Architectural Expert Queries" designed to stress-test these specific components. Each new entry will include:
- `query`: The user request.
- `expected_id`/`expected_title`: The primary target.
- `query_type`: A tag for filtering (e.g., `graph`, `visual`, `negative`).
- `ground_truth`: A short descriptive answer (not just the title) for better Ragas evaluation.

## 3. Query Categories & Examples

### A. Graph Multi-Hop (Neo4j)
Testing the ability to traverse multiple nodes in the knowledge graph.
- *Example:* "Quels sont les autres animés produits par le studio qui a fait Attack on Titan ?"
- *Target Component:* `neo4j_manager.get_enriched_context`, `deep_fact_checker`.

### B. Thematic Micro-Tags
Testing the alignment with the fine-grained tags generated by `DataIntelligence`.
- *Example:* "Cherche une œuvre avec une 'Esthétique Cyberpunk' et un 'Héros stoïque'."
- *Target Component:* `ChromaDB` thematic index + `DataIntelligence` tags.

### C. Visual Appearance (SigLIP)
Testing the vision-language mapping for character and poster descriptions.
- *Example:* "L'animé avec un personnage aux cheveux argentés portant un masque et lisant un livre."
- *Target Component:* `SigLIP-SO400M` embeddings, `VisionQuest`.

### D. Cross-Media Graph
Testing the linkage between different media types (Anime, Manga, Movies).
- *Example:* "Le film qui a été réalisé par l'auteur du manga Berserk."
- *Target Component:* `neo4j_manager` (Author -> [Manga, Movie]).

### E. Negative Cases & Ambiguity
Testing the robustness of the `Critic` agent and the `Searcher` planner.
- *Example:* "Est-ce qu'il existe un animé sur un chat qui devient un vaisseau spatial ?" (Expected: Negative/Not Found).
- *Target Component:* `critic_evaluation`, `searcher_plan`.

## 4. Implementation Plan
1. **Extend Schema**: Add optional `query_type` and `ground_truth` fields to `gold_dataset.json`.
2. **Add Queries**: Append ~50 manually crafted queries across the 5 categories.
3. **Update Evaluation Script**: Modify `src/pipeline/mlops/evaluation_metrics.py` to optionally filter by `query_type` and report per-category performance.
4. **Validation**: Run `rag_smoke_test.py` to ensure the new queries are integrated correctly.

## 5. Success Criteria
- The gold set covers at least 5 distinct architectural features.
- Evaluation metrics show a clear distinction between "Vector-only" and "Hybrid (Graph)" performance on multi-hop queries.
- Negative cases are correctly identified by the Critic agent.
